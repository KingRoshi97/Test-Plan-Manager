# PROJECT_OVERVIEW — 40 A.K.R.E.S. App (Canonical)

## 1) Mission (what are we building?)
40 A.K.R.E.S. is a mobile-first community platform for members to connect, learn, build, and govern together.
The app combines community chat (channels + DMs), live programming (rooms/livestream), governance (proposals + voting),
events, profiles, and ops/moderation tools under a role-based access model.
This repo is being rebuilt using the Roshi System to restore clarity, consistency, and production readiness.

## 2) Current rebuild context (why Roshi)
The app was close to production-ready before Roshi was added, but the repo became messy during the final push.
We are now using Roshi to rebuild cleanly from a single canonical overview + domain packs.
“Preexisting” means the code/docs that existed BEFORE Roshi was introduced.

## 3) Target users & roles (RBAC baseline)
- Visitor: onboarding role with minimal permissions. Can access public spaces/channels, view certain community pages, and manage own profile.
- Passport: verified member role with expanded community access (channels, posting, participation) based on rules in domain specs.
- Citizen: higher-trust member role with priority access and broader participation rights.
- Moderator: moderation capabilities (reports, mod actions, content controls) and elevated write permissions in constrained areas.
- Admin: full platform control (ops console, system settings, global permissions, audit review).

Notes:
- Public “Welcome” channel exists conceptually as an onboarding surface. Non-admins can read; write is restricted.
- Role rules must be expressed in Roshi domains (AccessControl + policies) and enforced consistently.

DB/schema/migrations remain unchanged regardless.

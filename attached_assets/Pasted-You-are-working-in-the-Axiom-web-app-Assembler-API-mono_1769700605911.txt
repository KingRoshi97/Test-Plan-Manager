You are working in the Axiom web app + Assembler API monorepo (Vite/React/TS client, Express/TS server, Drizzle/Postgres, object storage abstraction). We already have: Assemblies, Kits, Deliveries, Uploads, signing, adapters. We now need “PROJECT ZIP INTAKE → UPGRADE → OUTPUT PATCH/ZIP/PR” support.

GOAL
Let users upload a ZIP of an existing codebase (“Project Package”), provide a short upgrade request, and have Assembler produce:
1) Patch plan + diff (default, safest)
2) Optional patched zip (“upgraded_project.zip”)
3) Optional Git delivery (branch push) later

This must integrate into the existing Assembly/Kit/Delivery workflow and be included in the generated kit artifacts.

NON-NEGOTIABLES (SECURITY + RELIABILITY)
- Implement safe unzip (prevent zip slip / path traversal)
- Enforce limits: max files, max uncompressed bytes, max nested depth, timeout
- Quarantine/skip large binaries by default (warn)
- Store uploaded zip and extracted artifacts in object storage (Local for dev ok)
- Persist metadata in Postgres (Drizzle)
- Provide clear reason codes with ASSEMBLER_ prefix and correlation ids
- Do not break current API/routes/UI

2) STORAGE + HELPERS

- Function: safeUnzipZipToTemp(zipBufferOrPath, options) -> { extractedDir, stats, warnings }
- Must:
  - reject any entry with .. or absolute paths
  - enforce maxFiles (e.g. 20000)
  - enforce maxTotalUncompressedBytes (e.g. 250MB dev, configurable)
  - enforce maxEntrySize (e.g. 25MB)

  - detect binaries by extension and size; produce warning list

- Framework detection rules:
  - Next: next.config.*, app/ or pages/, next dependency

- Package manager:
  - pnpm-lock.yaml -> pnpm

C) Add server storage path conventions:

========================================================
3) API ENDPOINTS (v1)
========================================================
Add to server/v1-routes.ts

A) Create project package (upload zip)
POST /v1/project-packages
- multipart/form-data field: file (zip)
- Validate: content-type or extension .zip; max size
- Store original zip in object storage
- Compute sha256
- Create project_packages row with scanState=queued, indexState=queued
- Kick off background scan+index job (in-process queue)
- Response:
{
  "projectPackageId": "...",
  "scanState": "queued",
  "indexState": "queued",
  "warnings": [],
  "sha256": "..."
}

Reason codes:
- ASSEMBLER_UPLOAD_INVALID_TYPE
- ASSEMBLER_UPLOAD_TOO_LARGE
- ASSEMBLER_PROJECT_ZIP_INVALID
- ASSEMBLER_STORAGE_WRITE_FAILED

B) Get project package
GET /v1/project-packages/:projectPackageId
Return full status:
{
  "projectPackage": {
    "id": "...",
    "filename": "...",
    "sizeBytes": ...,
    "sha256": "...",
    "scanState": "...",
    "indexState": "...",
    "summary": {...},
    "warnings": [...],
    "error": { "code": "...", "message": "..." }?
  }
}

C) Attach project package to an assembly
POST /v1/assemblies/:assemblyId/project-packages/:projectPackageId/attach
- sets project_packages.assemblyId = assemblyId (if not already)
- also record in assembly inputJson (uploadIds/projectPackageId) if applicable
- audit log event

D) Upgrade request (generate patch artifacts in kit)
POST /v1/assemblies/:assemblyId/upgrade
Body:
{
  "projectPackageId": "...",
  "request": {
    "overview": "what they want done",
    "goals": ["..."],
    "constraints": ["..."],
    "doNotTouch": ["...optional globs..."]
  },
  "output": { "mode": "patch_only" | "patched_zip" | "git_pr_later" }
}

Behavior:
- Validate project package is indexed
- Create/Update kit upgrade artifacts:
  - project_summary.json (from indexer)
  - project_tree.txt
  - PATCH_PLAN.md (generated from request + summary)
  - patch.diff (initially may be empty if you only plan; still generate file)
  - If mode=patched_zip: generate upgraded_project.zip by applying patch (Phase 2)
- Response:
{
  "assemblyId": "...",
  "upgrade": {
    "mode": "patch_only",
    "artifacts": [
      {"type":"upgrade_plan","sha256":"...","sizeBytes":...},
      {"type":"upgrade_diff","sha256":"...","sizeBytes":...}
    ]
  }
}

NOTE: For Phase 1, implement “plan + diff placeholder” without auto-applying changes. Auto-apply comes in Phase 2 after adding a patch application engine.

Reason codes:
- ASSEMBLER_PROJECT_NOT_INDEXED
- ASSEMBLER_UPGRADE_REQUEST_INVALID
- ASSEMBLER_UPGRADE_GENERATION_FAILED

E) Fetch upgrade artifacts (optional convenience)
GET /v1/assemblies/:assemblyId/upgrade
- returns list of artifact signed URLs + checksums

4) BACKGROUND JOBS (in-process queue)
========================================================
Implement a minimal job runner (if you don’t already have one):
- server/jobs/queue.ts
- enqueue(name, payload)
- worker loop with concurrency=1-2
Jobs:
- scanAndIndexProjectPackage(projectPackageId)

scanAndIndexProjectPackage:
1) download zip from storage
2) safe unzip to temp
3) index project (summaryJson + artifacts)
4) upload artifacts to storage
5) update project_packages: scanState=scanned/indexState=indexed + summaryJson + warningsJson
6) on failure: set failed + errorCode/errorMessage

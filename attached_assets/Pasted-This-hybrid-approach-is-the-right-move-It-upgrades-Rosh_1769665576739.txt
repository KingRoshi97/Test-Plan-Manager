This hybrid approach is the right move. It upgrades Roshi from “prompt → invention risk” to “structured intent → deterministic expansion,” and it also makes your API + bundle handoff way more reliable.

Here’s the outline turned into a concrete product + API + bundle design you can implement immediately.

Run object schema (yours + add fields)

Store these under run.input (don’t spread into top-level unless you need indexing):

  runId: string

  step: RunStep
  input: {

Backward compatibility

Allow old payloads:

if user sends { idea: "..." }, you auto-map it into structured fields with:

projectName = "Untitled"

description = idea.slice(0, 140)

features/users empty
This keeps older clients working.

3) Engine change: “Structured Expansion” prompt contract

Instead of sending raw idea text, the engine should build a deterministic “AI context packet”:

AI_CONTEXT.json (generated per run)
{
  "projectName": "...",
  "description": "...",
  "features": [...],
  "users": [...],
  "techStack": {...},
  "preset": "..."
}


Then the AI expansion prompt becomes:

Use this structured packet as the primary truth

Never invent missing items; missing becomes UNKNOWN

Expand into:

PROJECT_OVERVIEW.md

domain packs (platform/api/web/infra/security)

action vocabulary + reason codes if needed

This will cut time and reduce hallucination because the “shape” is known upfront.

4) Template fallback (the reliability layer)

You already have generated templates. Make them the fallback in two cases:

When to fallback

AI expansion fails (timeout/error)

AI expansion returns invalid structure (missing required headers/tables)

Fallback behavior

Populate templates from structured inputs

Fill unknowns as UNKNOWN

Still produce bundle.zip + manifest + agent prompt

Mark in ROSHI_REPORT and manifest:

"generationMode": "ai" | "template_fallback"


This makes the system dependable.

5) Manifest upgrade for hybrid

Add a section so agents know what happened and what to trust:

{

Also embed the structured input inside the bundle:

handoff/input.json (raw structured input)

handoff/ai_context.json (normalized)

This is huge for vibecoding agents.

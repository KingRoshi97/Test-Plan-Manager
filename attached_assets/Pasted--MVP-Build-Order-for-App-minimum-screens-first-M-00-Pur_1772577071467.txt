## MVP Build Order for App (minimum screens first)

### M-00 Purpose

Define the minimal set of screens needed to operate AXION through the UI without building everything at once.

---

## M-01 MVP Phase 1 (can operate runs end-to-end)

### 1) Dashboard

**Must support**

* show latest run summary
* buttons: Doctor, Start Run, Advance Run
* view last action logs

**Depends on**

* `POST /api/doctor`
* `POST /api/run/start`
* `POST /api/run/advance`
* `GET /api/runs`

---

### 2) Runs List

**Must support**

* list runs
* open a run

**Depends on**

* `GET /api/runs`

---

### 3) Run Detail

**Must support**

* stage timeline
* open stage report
* open gate report (if present)
* open logs via pointers
* run stage (optional in MVP) or just advance

**Depends on**

* `GET /api/runs/:run_id`
* `GET /api/artifact`
* `GET /api/log`
* (optional) `POST /api/run/stage`

---

### 4) Gate Failures (basic)

**Must support**

* show failing gates
* show remediation
* open evidence pointers

**Depends on**

* `GET /api/runs/:run_id`
* `GET /api/artifact`
* `GET /api/log`

---

## M-02 MVP Phase 2 (verification + packaging)

### 5) Verify

**Must support**

* trigger verify
* show verification result
* show proof ledger pointer (if exists)
* show logs

**Depends on**

* `POST /api/verify`
* `GET /api/artifact`
* `GET /api/log`

---

### 6) Kits

**Must support**

* trigger pack
* show kit manifest + entrypoint
* show bundle export pointer + hash

**Depends on**

* `POST /api/pack`
* `GET /api/artifact`

---

## M-03 MVP Phase 3 (system introspection)

### 7) Registries (read-only)

**Must support**

* browse registries
* show schema validity (doctor output or inline)

**Depends on**

* `GET /api/artifact`
* (optional) `POST /api/doctor`

---

### 8) Proofs (ledger browser)

**Must support**

* browse ledger entries
* open proof objects and source logs

**Depends on**

* `GET /api/artifact`
* `GET /api/log`

---

## M-04 Hard boundaries (locked)

* MVP must not include “edit registries” or “run arbitrary commands”
* MVP must always operate by pointers and reports (repo truth)

If you want to continue: next is a **component inventory** for App (Layout, Nav, JSON viewer, log viewer, table, status badges) mapped to these MVP screens.

The Metric That Actually Matters
Generation Fidelity
Generation Fidelity = Correct Artifacts / Planned Artifacts

Where:

Planned Artifacts = files in the file_inventory

Correct Artifacts = files that:

compile

pass type checks

match expected file class

satisfy contracts/templates

pass verification tests

Why This Metric Matters

Imagine two builds:

Build A
files generated: 1200
compiles: yes
tests pass: yes

Fidelity:

~95%
Build B
files generated: 1200
compiles: partially
tests failing: many
contracts mismatched

Fidelity:

~55%

Both builds produced 1200 files, but only one is usable.

That’s why file count is a misleading success signal.

What You Should Track

For every build run, record:

Metric	Meaning
Planned files	file inventory size
Generated files	files written
Verified files	passed verification
Failed files	verification failures
Retry count	how many retries executed
Repair count	how many patch repairs

From that you compute:

Fidelity = verified_files / planned_files
Healthy Targets

For a system like Axion:

Stage	Expected Fidelity
Early prototype	50–65%
Stable system	70–85%
Mature system	90–95%

Anything above 90% means the build system is extremely reliable.

Why This Is the Real Signal

Axion is essentially a software compiler:

Agent Kit → Repo

Compilers are judged by:

correctness

not by how much code they emit.

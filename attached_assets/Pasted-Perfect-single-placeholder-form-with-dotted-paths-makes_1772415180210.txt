Rendering rules (v1, deterministic)

Given a selected template file and a “render context” object:

Find placeholders matching {{...}}

For each placeholder {{path.to.value}}:

resolve the dotted path against the context object

if value exists:

replace with string value

if missing:

leave placeholder unchanged (don’t delete it)

record it as unresolved in the report

Determinism rules

Replace placeholders left-to-right in the file

Do not reorder anything

Write output markdown with LF newlines

Write the render report with your canonical JSON rules (deep-sort keys, etc.)

Completeness scanner (v1)

Use a single regex:

Regex: \{\{\s*([a-zA-Z0-9_.-]+)\s*\}\}

That captures the placeholder key inside.

What counts as unresolved

A placeholder is unresolved if, after rendering, the output still contains a {{...}} token.

One decision you should lock right now

When a placeholder resolves to:

number / boolean → stringify as-is (true, 42)

array/object → JSON stringify (recommended) OR join arrays with ,

For MVP, safest is:

primitives → string

arrays/objects → JSON.stringify(value)

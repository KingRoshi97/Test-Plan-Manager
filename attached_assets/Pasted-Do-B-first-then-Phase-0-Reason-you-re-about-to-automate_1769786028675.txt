Controls:

Works for both warn + strict mode

Reason codes are readable and consistent

B2) Delivery UI: Safety gating visibility

Where: Deliveries tab / Delivery detail modal

UI requirements

If strict mode and there exists any BLOCK warnings related to delivery, show:

“Delivery blocked by Safety (STRICT)” badge

Link to Safety panel

“Override (admin)” button if you want (optional)

For warn mode:

Show “Warnings present” badge but allow delivery

Hook

On delivery attempts view, render:

last safety snapshot (count + max severity)

if blocked: show block reason code(s)

Definition of done

Anyone can understand why a delivery didn’t run or was blocked, without reading logs.

B3) Redacted download option (minimum viable)

You don’t need a full second “redacted kit.zip” yet. Keep it simple:

Option (fastest): Redacted compiled docs endpoint

GET /v1/assemblies/:id/inputs/redacted

returns a text/markdown file of USER_UPLOADS_COMPILED.md with secrets masked

Add a button in UI:

“Download redacted compiled uploads”

Why this is enough

The highest-risk thing is the compiled doc text or extracted samples containing secrets.

This gives safe sharing without overhauling the kit format.

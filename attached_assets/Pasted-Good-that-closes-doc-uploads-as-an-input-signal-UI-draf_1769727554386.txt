Implement Doc Ingestion v1 so uploaded docUploadIds affect the generated kit.

GOAL
When an assembly has docUploadIds:
- fetch each file from object storage
- extract text (pdf/docx/txt/md)
- generate normalized artifacts in the kit
- include references in manifest.json and PROJECT_OVERVIEW.md
- keep limits/safety

SERVER WORK
1) Create server/doc-ingestion.ts
- function ingestUploadedDocs(docUploadIds: string[]): Promise<{
    compiledMarkdownPath: string,
    perFileTextPaths: string[],
    stats: { files: number, chars: number }
  }>
- Use existing storage adapter to download bytes by fileId.
- Extract:
  - pdf: pdf-parse
  - docx: mammoth
  - md/txt: decode utf-8
- Normalize:
  - strip null bytes, collapse excessive whitespace
  - cap: max 1MB per file extracted text, max 4MB total
  - if over cap: truncate and note truncation in output
- Write outputs into the workspace temp folder used by kit packaging:
  - docs/inputs/USER_UPLOADS_COMPILED.md
  - docs/inputs/uploads/<fileId>.txt

2) Update pipeline run (where you build assembly input / kit workspace)
- If docUploadIds exist, call ingestUploadedDocs
- Ensure those files are included in kit.zip packaging
- Add to manifest.json:
  inputArtifacts: [
    "docs/inputs/USER_UPLOADS_COMPILED.md",
    ...perFileTextPaths
  ]
  inputStats: { files, chars }

3) Update PROJECT_OVERVIEW.md generation
- Insert section:
  "Uploaded Docs Snapshot"
  - file count
  - bullet list of filenames if available
  - link paths in the kit
- Also inject a short summary paragraph into mode overlay context if mode != new_build.

CLIENT (OPTIONAL NICE)
- On Review step, show:
  "Docs uploaded: N" + list of filenames

SAFETY
- Reject unknown mime types
- enforce size limits early
- never execute anything from uploaded docs
- log reason codes when extraction fails but continue with others

ACCEPTANCE
- Create assembly with doc uploads produces kit that contains docs/inputs/USER_UPLOADS_COMPILED.md
- manifest.json lists the inputArtifacts
- PROJECT_OVERVIEW.md includes Uploaded Docs Snapshot
- pipeline still succeeds if one upload fails (marks partial)

# VER-0a — Boundary Checklist

Belongs in `verification/` if it answers:
- "What proofs are required and what do they look like?"
- "How do we record command execution and results?"
- "How do we decide when work/run is complete?"

Does NOT belong in `verification/` if it answers:
- "How do we evaluate gate predicates?" (gates)
- "What standards require proofs?" (standards)
- "How do we package the kit?" (kit)

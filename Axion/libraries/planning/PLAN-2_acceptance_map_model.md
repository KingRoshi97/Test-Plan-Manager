# PLAN-2 — Acceptance Map Model

## Purpose
Define a structured map from requirements to:
- acceptance criteria
- artifacts that must be produced
- proofs/verification that must exist

This enables coverage checks in planning and verification gates.

## Structure
- requirement nodes (from canonical spec)
- acceptance criteria (testable statements)
- evidence requirements:
  - artifact contract IDs
  - proof ledger requirements

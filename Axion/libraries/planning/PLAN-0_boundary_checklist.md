# PLAN-0a — Boundary Checklist

Belongs in `planning/` if it answers:
- "How do we break work into tasks?"
- "How do we map requirements to acceptance criteria?"
- "How do we sequence tasks deterministically?"
- "What does plan coverage mean?"

Does NOT belong in `planning/` if it answers:
- "What templates exist?" (templates)
- "How do we evaluate gates?" (gates)
- "What is the canonical schema?" (canonical)

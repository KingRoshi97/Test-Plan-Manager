# KIT-0a — Boundary Checklist

Belongs in `kit/` if it answers:
- "What does the output bundle look like?"
- "What files must be included and how are they referenced?"
- "How do we export safely (internal vs external)?"

Does NOT belong in `kit/` if it answers:
- "How do we select templates?" (templates)
- "What are the proofs?" (verification)
- "How do we evaluate gates?" (gates)

# Ableton Live Production Readiness Checklist

## Summary
This checklist ensures your Ableton Live project is production-ready for export, performance, or collaboration. By following these steps, you’ll optimize your workflow, avoid technical issues, and maintain professional standards. This guide covers project organization, audio quality, plugin management, and export settings.

## When to Use
- Before exporting a track for final mastering or distribution.
- Prior to live performance using Ableton Live.
- When handing off a project to collaborators or clients.
- Before archiving a project for long-term storage.

## Do / Don't

### Do:
- **Do organize tracks and groups.** Use meaningful names and group related tracks for clarity.
- **Do freeze and flatten tracks with CPU-intensive plugins.** This reduces processing load and ensures compatibility.
- **Do verify audio levels.** Ensure no clipping occurs and maintain headroom for mastering (-6 dB recommended).

### Don't:
- **Don't leave unused tracks or clips.** Remove them to avoid confusion and reduce file size.
- **Don't rely on external samples or plugins without consolidation.** Missing assets can break the project for collaborators.
- **Don't export without checking sample rate and bit depth settings.** Incorrect settings can compromise audio quality.

## Core Content

### 1. **Project Organization**
- **Rename tracks clearly.** Use descriptive names like "Kick," "Bassline," or "Lead Synth."
- **Group related tracks.** For example, group all drum tracks under a "Drums" group.
- **Color-code tracks.** Assign colors to track types for visual clarity.

### 2. **Audio Quality and Levels**
- **Check for clipping.** Ensure all tracks peak below 0 dB. Use a limiter on the master channel if needed.
- **Maintain headroom.** Keep master output levels around -6 dB to allow space for mastering.
- **Remove unwanted noise.** Use EQ or noise gates to clean up tracks.

### 3. **Plugin and Sample Management**
- **Freeze and flatten tracks.** For CPU-heavy plugins or instruments, freeze and flatten to audio to ensure smooth playback and compatibility.
- **Consolidate external samples.** Use "Collect All and Save" to ensure all samples are included in the project folder.
- **Check plugin versions.** Verify that collaborators or performance systems have compatible plugin versions.

### 4. **Export Settings**
- **Set sample rate and bit depth.** Use 44.1 kHz/16-bit for standard audio or 48 kHz/24-bit for high-quality production.
- **Enable dithering.** If exporting to a lower bit depth, use Ableton’s dithering options to minimize quantization noise.
- **Export stems.** For collaboration or remixing, export individual tracks as stems.

### 5. **Backup and File Management**
- **Save incremental versions.** Use versioned filenames (e.g., `ProjectName_v2.als`) to track changes.
- **Create a backup.** Store a copy of the project on an external drive or cloud storage.
- **Test the project.** Open the project on another system to ensure all files and plugins load correctly.

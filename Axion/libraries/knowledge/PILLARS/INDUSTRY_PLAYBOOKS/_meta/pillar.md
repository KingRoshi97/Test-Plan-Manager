# INDUSTRY_PLAYBOOKS Pillar

## Purpose

The Industry Playbooks pillar provides domain-specific knowledge items organized by industry vertical. Each industry contains standardized knowledge categories covering workflows, data models, integrations, compliance, security/risk, KPIs, and templates guidance.

## Industries

1. **Healthcare** — Patient care systems, EHR/EMR, HIPAA compliance, claims processing
2. **Finance (Banking/Payments)** — KYC, transactions, fraud detection, PCI compliance
3. **Retail & Ecommerce** — Shopping workflows, inventory, fulfillment, marketplace integrations
4. **Logistics & Supply Chain** — Warehouse operations, carrier integrations, EDI, shipment tracking
5. **Government / Public Sector** — Case management, benefits eligibility, records requests, inter-agency data sharing

## Structure

Each industry folder contains:
- `_meta.md` — Industry overview and context
- `workflows/` — End-to-end workflow maps
- `data_models/` — Entity maps and data classification
- `integrations/` — Integration patterns and vendor maps
- `compliance/` — Regulatory and audit requirements
- `security_risk/` — Threat models and abuse patterns
- `kpis_metrics/` — Operational KPIs and reliability targets
- `templates_guidance/` — Template and standards pack recommendations

## Selection Rules

- Filter by industry tag matching canonical spec signals
- Filter by executor access level
- Apply maturity threshold based on risk class
- Rank by domain relevance

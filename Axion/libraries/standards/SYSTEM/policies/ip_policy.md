# Intellectual Property Policy — Standards Library

## Purpose

Defines IP rules for standard unit content, including authorship, reuse, and attribution requirements.

## Core Rules

### Authorship

- All standard units are authored internally and owned by the standards library owner
- Standard units must NOT contain third-party copyrighted content without license verification
- Example sections must use clearly synthetic/placeholder values

### Reuse

- Standard rules may be freely applied as constraints during builds and reviews
- Verbatim copying of standard rule text into outputs requires citation
- Standard verification procedures may be reformatted for context but must preserve meaning

### Secrets and Sensitive Data

- Standard units must NOT contain actual secrets, API keys, passwords, or credentials
- Standard units must NOT contain real PII
- Security standards referencing sensitive patterns should describe the pattern abstractly

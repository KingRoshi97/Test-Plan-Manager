## 2. Purpose

Define the canonical rules for frontend data binding: which screens/components call which
endpoints, when calls happen (on mount, on focus, on user action), cache usage, invalidation
rules, and error handling bindings. This template must be consistent with state management
strategy and API endpoint specs and must not invent endpoint IDs or call patterns not present in
upstream inputs.

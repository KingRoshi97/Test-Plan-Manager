## 2. Purpose

Define the canonical accessibility (a11y) implementation requirements across the frontend,
including semantic roles, labels, focus management, keyboard navigation, screen reader
announcements, color contrast considerations, and enforcement/testing practices. This template
must be consistent with UI composition patterns and must not invent platform requirements not
present in upstream inputs.

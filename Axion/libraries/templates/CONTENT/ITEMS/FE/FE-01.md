## 2. Purpose

Define the canonical route map and layout structure for the frontend: all routes/screens, their
layout shells, navigation placement, required guards, and layout composition rules. This
template must be consistent with route contracts and must not invent route IDs or screen IDs
not present in upstream inputs.

## 11. Completeness Gate

- All Required Fields must be populated or explicitly marked UNKNOWN with justification.

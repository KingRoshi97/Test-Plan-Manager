Purpose
Define enforceable rules for accessible error communication across forms and system errors:
how errors are presented, announced, linked to fields, and recoverable. This ensures errors are
understandable and operable for keyboard and screen reader users.

Rules
   ‚óè‚Äã Every error must answer: what happened, what it affects, what to do next.
   ‚óè‚Äã Errors must not rely on color alone; must include text and semantic markup.
   ‚óè‚Äã Inline errors must be announced when they appear or when focus enters the field

   ‚óè‚Äã If error summary exists, it must be keyboard navigable and link to each error field.
   ‚óè‚Äã Permission errors must not leak restricted info.

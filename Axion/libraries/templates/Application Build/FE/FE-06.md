## 2. Purpose

Define the canonical integration rules between the frontend implementation and the design
system tokens: token sources, naming, usage rules, theme switching, component token binding,
and enforcement checks. This template must be consistent with design system docs and must
not invent token names or theme mechanics not present in upstream inputs.

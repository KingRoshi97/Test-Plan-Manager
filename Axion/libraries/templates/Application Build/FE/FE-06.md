## 2. Purpose

Define the canonical catalog of loading, empty, and error states for all screens and key components: skeleton/placeholder patterns, empty-state messaging, error display patterns, retry policies, and fallback behaviors. This template ensures every screen has defined states for all data-fetching outcomes.

## 2. Purpose

Define the canonical UX rules for surfacing and recovering from errors in the client: how
different error classes map to UI surfaces (inline, toast, modal, full-screen), what copy to show,
what actions to offer (retry, re-auth, report), and how to keep behavior consistent across screens
and components. This template must be consistent with API error policy and client recovery
strategies and must not invent error classes not present in upstream inputs.

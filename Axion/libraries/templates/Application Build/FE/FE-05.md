## 2. Purpose

Define the canonical theming and design-token integration for the frontend: token categories, naming conventions, theme variants, component token bindings, override/customization policy, and dark/light mode support.

- Must align to: {{standards.rules[STD-CANONICAL-TRUTH]}} | OPTIONAL

- Components SHOULD bind to tokens, not raw values, where possible.

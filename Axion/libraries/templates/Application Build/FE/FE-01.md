# FE-01 — Route Map + Layout Spec

## 1. Header Block

| Field             | Value                                              |
|-------------------|----------------------------------------------------|
| Template ID       | FE-01                                            |
| Template Type     | Build / Frontend                                 |
| Template Version  | 1.0.0                                            |
| Applies           | All projects requiring route map + layout spec   |
| Filled By         | Internal Agent                                   |
| Consumes          | Canonical Spec, Standards Snapshot               |
| Produces          | Filled Route Map + Layout Spec                   |

## 2. Purpose

Create the canonical inventory of all screens/views in the product, including screen_id, route, layout_ref, auth requirements, and the feature(s) each screen serves. This template drives all downstream frontend templates and ensures every feature in the product has an identifiable screen.

## 6. Rules

- Must align to: {{standards.rules[STD-CANONICAL-TRUTH]}} | OPTIONAL
- Every screen_id MUST be unique.
- Every feature_id in PRD-01 SHOULD bind to at least one screen (or be flagged as UNKNOWN).
- Do not introduce routes not implied by upstream feature/PRD inputs.
- If a required field is not provided by inputs, write UNKNOWN unless explicitly disallowed.
- Use consistent terminology from: {{glossary.terms}} | OPTIONAL

## 10. Completeness Gate

- Gate ID: TMP-05.PRIMARY.FE
- Pass conditions:
  - [ ] required_fields_present == true
  - [ ] all screen_ids are unique
  - [ ] every feature_id has at least one screen (or flagged)
  - [ ] placeholder_resolution == true
  - [ ] no_unapproved_unknowns == true

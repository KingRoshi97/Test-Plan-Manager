## 2. Purpose

Define the canonical UI state model used across screens and components: the required state
set (loading/empty/error/success), how transitions happen, what content appears in each state,
and consistency rules for retries and recoveries. This template must be consistent with client
error handling strategy and must not invent state types not present in upstream inputs.

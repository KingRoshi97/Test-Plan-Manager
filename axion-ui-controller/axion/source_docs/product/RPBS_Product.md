# Requirements and Product Behavior Specification (RPBS)

**Project:** axion-ui-controller
**Version:** 0.1.0
**Status:** DRAFT

## Product Overview

AXION UI Controller is a web-based dashboard for managing and executing AXION documentation pipeline stages. It provides a visual interface to create kits, run pipeline stages (docs:scaffold, docs:content, app:bootstrap), monitor progress, and review generated artifacts. The goal is to replace CLI-only usage with an intuitive UI that guides users through the documentation-first development workflow.

## Target Users

- **Developers** who want to use AXION to scaffold new projects with proper documentation
- **Technical leads** who need to review and approve generated documentation before locking
- **Solo founders** building MVPs who want structured documentation without manual effort

## Core Features

1. **Kit Creation Wizard** - Create new AXION kits with project name, stack profile selection, and initial RPBS/REBS input
2. **Pipeline Dashboard** - View all active kits, their current stage status, and progress through the pipeline
3. **Stage Runner** - Execute individual stages (generate, seed, draft, review, verify, lock) or run full plans (docs:full, app:bootstrap)
4. **Artifact Viewer** - Browse and read generated documentation, templates, and code files
5. **Stage Markers View** - Visual representation of stage_markers.json showing completed/pending stages per module
6. **Override Controls** - Toggle override mode for bypassing quality gates during development
7. **Console Output** - Real-time display of AXION script output and logs

## User Stories

**US-01: Create New Kit**
As a developer, I want to create a new AXION kit by entering a project name and selecting a stack profile, so I can start the documentation pipeline.

**US-02: Fill RPBS/REBS**
As a developer, I want to enter my product requirements and engineering standards in a form, so AXION can generate accurate documentation.

**US-03: Run Pipeline Stage**
As a developer, I want to click a button to run a specific pipeline stage (e.g., docs:scaffold), see real-time output, and know when it completes.

**US-04: View Stage Progress**
As a developer, I want to see which modules have completed which stages, so I understand the overall pipeline state.

**US-05: Review Generated Docs**
As a technical lead, I want to browse and read generated documentation files, so I can verify quality before locking.

**US-06: Override Gates**
As a developer, I want to enable override mode to bypass verify failures during rapid iteration, understanding this skips quality checks.

## Business Rules

- **BR-01**: A kit must have RPBS and REBS filled before running docs:scaffold
- **BR-02**: docs:content requires docs:scaffold to complete first
- **BR-03**: app:bootstrap requires at least draft stage for all modules
- **BR-04**: Override mode must show clear warning that quality gates are bypassed
- **BR-05**: Lock stage cannot be overridden unless explicitly enabled in config

## Success Metrics

- **M-01**: Users can create a kit and run full pipeline in under 10 minutes
- **M-02**: 90% of users complete pipeline without needing CLI fallback
- **M-03**: Generated documentation passes verify stage on first attempt for 70% of runs
